# 算法 / Algorithm

这里不是只放题解，而是沉淀算法思维。

重点不是“这题答案是什么”，而是：

```text
我如何识别题目结构？
我为什么没想到？
下次看到什么信号要联想到什么方法？
```

## 核心链路 / Core Flow

```text
题目 → 边界 → 暴力解 → 瓶颈 → 状态 → 不变量 → 优化 → 复盘
```

## 目录 / Structure

```text
templates/   题目复盘模板、题型总结模板
patterns/    算法模式：滑动窗口、前缀和、DFS/BFS 等
problems/    具体题目复盘
```

## 高屋建瓴的思考流程 / High-level Thinking Flow

```text
1. 这题的对象是什么？数组、字符串、树、图、操作序列？
2. 答案空间是什么？暴力怎么枚举？
3. 它有没有关键词：连续、最短、最长、路径、方案数、频率、最小最大？
4. 暴力慢在哪里？
5. 我需要维护什么状态？
6. 有没有单调性可以让我不回头？
7. 有没有 HashMap / 栈 / 队列 / 堆 / DP 数组能保存“有用历史”？
8. 我的不变量是什么？
```

## 最重要的一句话 / Key Sentence

```text
枚举答案空间 → 找重复/瓶颈 → 维护有用状态 → 利用单调性或结构减少搜索
```
