# 软件工程 / Software Engineering

这里沉淀真实开发中的工程判断。

重点不是记录技术名词，而是记录：

```text
问题边界是什么？
为什么这样设计？
风险在哪里？
怎样算完成？
下次遇到类似问题如何判断？
```

## 重点主题 / Topics

- 边界 / Boundary
- 需求拆解 / Requirement Breakdown
- 测试 / Testing
- 架构 / Architecture
- 交付 / Delivery
- 代码质量 / Code Quality
- 技术选型 / Technical Decision
- 线上问题复盘 / Incident Review

## 常用问题 / Useful Questions

```text
1. 这个问题属于哪个领域？
2. 它不属于什么范围？
3. 当前版本需要做到什么程度？
4. 哪些是必须交付，哪些是后续优化？
5. 如果这样设计，风险是什么？
6. 怎么验证它真的完成了？
```
