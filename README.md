# notes

我的个人沉淀仓库 / Personal Knowledge Repository

这个仓库用于长期整理我的学习、思考、算法、工程经验、观点判断和文章输出。

它不是单纯的资料收藏夹，也不是只放代码的仓库。它的核心目标是：

```text
输入 → 思考 → 结构化 → 复盘 → 沉淀 → 输出
```

## 核心原则 / Core Principles

- 不只记录结论，也记录思考路径
- 不只保存答案，也保存为什么当时没想到
- 不把经验绝对化，始终写清边界
- 从零散输入中提炼可迁移的判断框架
- 中文为主，关键标题和概念附英文，顺便积累英语表达

## 目录结构 / Structure

```text
00-inbox/                 临时收集区
01-thinking/              思考框架、逻辑、认知、边界
02-algorithm/             算法题、题型模式、复盘模板
03-software-engineering/  软件工程、架构、测试、交付
04-opinions/              观点、视频、评论区言论拆解
05-articles/              文章草稿与完成稿
06-reading/               阅读、资料、课程、书籍
99-meta/                  仓库规则、AI 协作说明、写作规则
```

## 使用方式 / How to Use

先把没整理好的内容放进 `00-inbox/`。

每隔一段时间，把 inbox 里的内容移动到对应模块，并补充：

- 背景是什么？
- 我的第一反应是什么？
- 背后的结构是什么？
- 边界在哪里？
- 这件事可以沉淀成什么框架？

## 整理目标 / Goal

这个仓库真正要沉淀的不是知识点数量，而是：

- 判断力 / Judgment
- 思考方式 / Thinking Method
- 工程经验 / Engineering Experience
- 算法识别能力 / Algorithm Pattern Recognition
- 表达能力 / Expression
